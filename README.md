# KLM ESP32 IOT模块 项目

## 项目概述

这是一个基于ESP32的咖啡机IoT联网模块项目，为咖啡机提供完整的智能化联网功能。项目采用模块化设计，支持MQTT通信、OTA固件升级、蓝牙配网、透传模式、简化协议处理和AT指令等功能，实现咖啡机的远程控制、状态监控、配方管理和设备维护等IoT应用。

目前支持C3、S3两种模块，主分支是ESP32-C3，S3分支对应ESP32-S3模块，需要注意后续建议使用S3模块，才能正常使用mqtts https以及蓝牙加密功能。

特别注意，目前只有ESP32-S3模块的程序通过了TUV认证，海外版需要使用S3。

备注：本项目已经有过一次框架变更，引入简化协议使很多事情的处理核心放到了IOT模块上。例如原来的蓝牙通信由MCU自行处理，现在IOT模块直接与小程序通信。详细参考架构对比部分。

## 开发环境

### 环境要求

- **ESP-IDF 版本**: 5.2.1 (一般不建议版本迁移，除非有特殊需求新版本IDF)
- **目标芯片**: 
  - ESP32-C3 (主分支)
  - ESP32-S3 (S3分支，推荐后续使用，资源充足有加密)
- **开发工具**: Visual Studio Code + ESP-IDF 扩展

### 开发环境配置

1. **配置 VSCode**
   - 安装 ESP-IDF 扩展
   - 配置 ESP-IDF 路径和工具链路径

2. **编译项目**
   - 依据当前所在分支设置目标芯片为 ESP32-C3 或 ESP32-S3
   - IDF插件中选择编译

3. **烧录和监控**
   - IDF插件中选择烧录
   - 监控串口输出
   - 也可以直接点击插件中的一键编译、烧录和监控

### 调试配置

#### 内置USB JLINK调试 - 不好用，经常连不上

ESP32-C3和ESP32-S3都支持内置的USB JTAG调试功能，无需外部调试器：

1. **硬件连接**
   - 使用USB数据线连接开发板的USB接口到电脑
   - 确保开发板上的USB JTAG功能已启用

2. **VSCode配置**
   - 在ESP-IDF扩展中配置调试器类型为"ESP USB Bridge"
   - 设置调试端口（通常自动检测）


#### 外置JLINK调试  - 在S3上成功连接，比较稳定

对于需要更专业调试功能的场景，可以使用外置JLINK调试器：

1. **硬件要求**
   - JLINK调试器（如JLINK EDU、JLINK PLUS等）
   - 连接线缆（根据开发板接口类型选择）

2. **连接方式**
   - 将JLINK调试器连接到ESP32的JTAG接口
   - 具体连接方法和配置请参考：[ESP32-S3 Jlink调试文档](doc/ESP32-S3%20Jlink调试.pdf)

### 模块概览

本项目采用模块化设计，主要包含以下核心模块：

1. **主控制模块** - 系统初始化、事件驱动状态机、数据路由分发
2. **BSP驱动模块** - WiFi、MQTT、BLE、UART、TCP、LED等硬件驱动
3. **OTA升级模块** - 外部MCU固件升级，支持YMODEM协议传输
4. **AT指令模块** - 统一的AT指令解析与执行框架
5. **简化协议模块** - JSON消息处理，多版本协议支持
6. **MQTT认证模块** - 动态MQTT认证（暂未启用）
7. **通用工具模块** - 环形缓冲、调试TCP等工具组件
